
# Qwen Safety Classifier v3 unsloth can do step qwen3 good

Fine-tuning **Qwen3.5-0.8B-Instruct** with **QLoRA** for Indonesian content safety classification under UU ITE Pasal 28 Ayat 2.

- **Input**: image or video frame  
- **Output**: detailed reasoning in Indonesian + label `SAFE` / `UNSAFE`  
- **Client**: Indonesian Government

---

## Dataset Structure

```
dataset/
├── metadata.csv              ← source of truth
└── Kontent/
    ├── class_0/
    ├── class_1/
    ├── class_2/
    ├── class_3/
    └── class_4/
```

### metadata.csv format

```
image_name,CLASSIFICATION,kategori,REASONING
1_126.jpg,UNSAFE,HENTAI,"Berdasarkan ..."
```

### Image naming convention

`X_Y.jpg` → file `Y.jpg` inside `Kontent/class_X/`

Example: `1_126.jpg` → `dataset/Kontent/class_1/126.jpg`

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

For Unsloth (Kaggle/Colab CUDA 12.1):

```bash
pip install "unsloth[colab-new] @ git+https://github.com/unslothai/unsloth.git"
```

### 2. (Optional) Extract video frames

If your dataset contains video files inside the class folders:

```bash
python 00_extract_frames.py --fps 1
```

### 3. Prepare dataset

```bash
# Verify all images exist on disk before writing JSON
python 01_prepare_dataset.py --verify
```

This reads `dataset/metadata.csv`, resolves paths using the `X_Y.jpg` convention, and writes:
- `dataset/train_data.json` (80%)
- `dataset/val_data.json` (20%)

### 4. Train

**Option A — Standard TRL (~6GB VRAM)**

```bash
python train_trl.py
```

**Option B — Unsloth (~3GB VRAM, 2-2.7x faster)**

```bash
python train_unsloth.py
```

Both scripts read all hyperparameters from `config.yaml`.

### 5. Evaluate

```bash
python eval.py --checkpoint output/trl_checkpoint/final_adapter
python eval.py --checkpoint output/unsloth_merged --merged
```

## Model Details

| Parameter | Value |
|-----------|-------|
| Base model | Qwen3.5-0.8B-Instruct |
| Method | QLoRA (4-bit base + 16-bit adapters) |
| LoRA rank | 8, alpha 16 |
| Target modules | q_proj, v_proj, up_proj, down_proj |
| Thinking mode | Off (direct output) |
| Max new tokens | 512 |
| Output language | Indonesian |

---

## Metrics (Priority Order)

| Metric | Type | Why |
|--------|------|-----|
| **Recall (UNSAFE)** | Classification | Priority — unsafe content must not pass through |
| F1-Score | Classification | Handles class imbalance |
| AUC-ROC | Classification | Threshold-independent |
| BERTScore | Reasoning quality | Indonesian text evaluation |
| Train/Val Loss | Training health | Detect overfitting |

**Recall > Precision** — a false alarm is better than missing unsafe content.

## Configuration

All hyperparameters are in `config.yaml`. Key sections:

```yaml
model:
  name: "Qwen/Qwen3.5-0.8B-Instruct"

lora:
  r: 8
  alpha: 16
  target_modules: [q_proj, v_proj, up_proj, down_proj]

training:
  learning_rate: 2.0e-4
  num_train_epochs: 3
  per_device_train_batch_size: 4

dataset:
  metadata_csv: "dataset/metadata.csv"
  image_root:   "dataset/Kontent"
  class_folders:
    0: "class_0"
    1: "class_1"
    ...
```
